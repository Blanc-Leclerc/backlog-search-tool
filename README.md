# Backlog 検索ツール (Backlog Search Tool)

複数のBacklogダッシュボード（スペース）やプロジェクトを横断して、高速にチケットを検索・分析できるツールです。
公式機能では難しい「ダッシュボードを跨いだ検索」や「AI分析用のデータ出力」を強力にサポートします。

## 🌟 特徴 (Features)

*   **ダッシュボード横断検索**: 複数のBacklogスペース（例: `company-a.backlog.com` と `company-b.backlog.com`）をまとめて検索できます。これが最大の強みです！
*   **爆速ローカル検索**: データをブラウザのローカルDB（IndexedDB）にキャッシュするため、ネットワーク遅延なしでサクサク検索できます。
*   **AIフレンドリー**: 検索結果をAI（ChatGPTなど）が理解しやすい形式でJSON出力したり、CSVで保存したりできます。
*   **プライバシー重視**: データはすべてお使いのブラウザ内に保存され、外部サーバーには送信されません。
*   **多言語・ダークモード対応**: 日本語/英語、ライト/ダークモードを完備。

## 🚀 使い方 (Usage)

1.  **初期設定**:
    *   `index.html` を開き、設定画面へ移動します。
    *   BacklogのスペースURL（Host）とAPIキーを入力して登録します。
    *   検索対象にしたいプロジェクトを選択して保存します。
2.  **データの同期**:
    *   「🔄 データの最新化」ボタンを押すと、Backlogからチケット情報を取得してローカルに保存します。
    *   初回は時間がかかりますが、2回目以降は差分更新なので高速です。
3.  **検索**:
    *   キーワード、担当者、状態などでフィルタリングできます。
    *   結果行をクリックすると詳細が表示されます。
4.  **活用**:
    *   「💾 AI用データ保存」でデータをダウンロードし、LLMに分析させることができます。
    *   「📊 CSV保存」でExcelなどで開ける形式でダウンロードできます。

## 📦 インストール (Install)

特別なインストールは不要です。フォルダ内の `index.html` をブラウザ（Chrome/Edge推奨）で開くだけで動作します。

## 🛠️ 技術スタック

*   HTML5 / CSS3 / JavaScript (Vanilla JS)
*   IndexedDB (Local Database)
*   Web Workers (Background Processing)
*   Chart.js (Analysis)
